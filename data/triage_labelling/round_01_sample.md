# Triage labelling — round 01 (30 applications)

Sample composed from the ingested PlanIt universe (1,549 applications, 2018-01 → 2026-05).

**How to fill this in:**

- For each application, look only at what's shown here (description + metadata + the source link if you need to dig). That's the same surface area the Stage 1 triage prompt will work from.
- Replace the `____` placeholders inline. Don't overthink — a one-line *why* is the most valuable thing here, even when the verdict feels obvious.
- Be honest with the **Confidence** and **Verify with Aisha?** columns. Anything where you'd lean one way but can imagine her disagreeing → flag for verification.
- Aim for ~1.5–2 minutes per application. Total ~60–75 minutes; OK to split across sittings.

**Signal-tag starter list** (tick all that apply; add new ones as `[other: name]`):

`substation` · `generator` · `backup` · `fuel storage` · `energy centre` · `gas` · `LPG` · `CHP` · `turbine` · `grid connection` · `cooling` · `air cooling` · `water cooling`

**Verdict scheme:**
- **DC** — this is a new data centre application (or substantially DC-focused).
- **adjacent** — DC-related but not a new DC itself (extension, infrastructure, supporting kit, conditions discharge on an underlying DC).
- **unrelated** — the 'data centre' keyword matched but the application isn't really about a DC.
- **unknown** — not enough information to decide; needs documents.
